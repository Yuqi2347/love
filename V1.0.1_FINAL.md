### 6. 朋友圈与发现模块分离

**设计说明：**
- **朋友圈 (`post_type='TIMELINE'`)**：只有互相关注的用户可以看到

**实现方式：**

- 后端：新增 POST /feed/discovery 接口发布发现模块帖子
- 前端：DiscoverView.vue 调用 createDiscoveryPost 发布

### 7. 关注/粉丝列表功能

**实现方式：**
- 前端：ProfileView.vue 添加关注/粉丝列表弹窗
- 始终显示当前查看用户的关注/粉丝数量（不是当前登录用户的）
- 所有用户都可以查看自己的关注/粉丝
- 只有管理员可以查看其他用户的关注/粉丝列表
- 点击用户卡片跳转到该用户的主页，显示完整的匹配分析等信息
- 聊天按钮（暂时提示"功能开发中"）

**后端接口：**
- `GET /follow/following` - 获取当前登录用户的关注列表
- `GET /follow/followers` - 获取当前登录用户的粉丝列表
- `GET /follow/user/{userId}/following` - 获取指定用户的关注列表
- `GET /follow/user/{userId}/followers` - 获取指定用户的粉丝列表

**前端实现：**
```typescript
// 加载用户资料时，始终加载该用户的关注/粉丝数量
async function loadProfile() {
  // ...
  const [fing, fers] = await Promise.all([
    isMe.value ? getFollowingList() : getUserFollowing(profileId.value),
    isMe.value ? getFollowerList() : getUserFollowers(profileId.value)
  ])
  followingCount.value = fing.data.data?.length || 0
  followerCount.value = fers.data.data?.length || 0
}

// 打开关注列表时检查权限
function handleOpenFollowing() {
  const isAdmin = userStore.user?.isAdmin || false
  if (!isMe.value && !isAdmin) {
    ElMessage.warning('只有管理员可以查看他人的关注列表')
    return
  }
  showFollowing.value = true
}

// 监听弹窗打开，加载对应的数据
watch(showFollowing, async (val) => {
  if (val) {
    const res = isMe.value ? await getFollowingList() : await getUserFollowing(profileId.value!)
    followingList.value = res.data.data || []
  }
})
```

## 问题修复说明

### 修复1：管理员账号显示 Lv1 问题

**原因：** `/feed/level-info` 接口传的是 null，没有从 CurrentUser 获取当前用户ID

**修复：**
```java
// ActivityService.java - getUserLevelInfo 方法
public UserLevelInfo getUserLevelInfo(Long userId) {
    if (userId == null) {
        userId = CurrentUser.getId();  // 获取当前登录用户ID
    }
    ...
}
```

### 修复2：登录后等级信息不完整

**原因：** 登录响应不包含 isAdmin 和 userLevel，前端缓存了不完整数据

**修复：**
```typescript
// userStore.ts - setAuth 改为 async，登录后立即获取完整profile
async function setAuth(auth: AuthResponse) {
  // ... 设置基本信息
  await fetchProfile() // 立即获取完整用户信息
}
```

```java
// UserService.java - toProfileResponse 方法添加null处理
.activityScore(user.getActivityScore() != null ? user.getActivityScore() : 0)
.userLevel(user.getUserLevel() != null ? user.getUserLevel() : 1)
.isAdmin(user.getIsAdmin() != null ? user.getIsAdmin() : false)
```

### 修复3：头像上传"系统内部错误"

**原因1：** 上传目录使用相对路径 `./uploads/` 导致路径解析错误

**修复：** 修改 application.yml 使用绝对路径
```yaml
app:
  upload:
    path: D:/study_resources/Githubpro/Love/campus-love-backend/uploads/
```

**原因2：** 静态资源未映射导致上传后无法访问

**修复：** 新增 FileUploadConfig.java 配置静态资源映射
```java
registry.addResourceHandler("/uploads/**")
        .addResourceLocations("file:" + uploadPath);
```

**原因3：** Vite 代理配置不正确

**修复：** 修改 vite.config.ts
```typescript
'/uploads': {
  target: 'http://localhost:8080',
  changeOrigin: true,
  rewrite: (path) => path.replace(/^\/uploads/, '/api/uploads'),
}
```

**原因4：** 头像 CSS 缺少 object-fit

**修复：** ProfileView.vue 添加样式
```scss
.avatar {
  width: 100px;
  height: 100px;
  border-radius: 50%;
  object-fit: cover;  // 确保图片按比例裁剪
}
```

### 修复4：发现页面等级显示问题

**原因：** 发现页面没有始终显示等级信息，只在等级不足时显示

**修复：** DiscoverView.vue 修改头部布局，始终显示等级信息

### 修复5：朋友圈和发现模块内容混淆

**原因：** 之前的实现没有区分帖子类型，所有帖子都会在两个模块显示

**修复：**
1. 添加 post_type 字段区分 TIMELINE/DISCOVERY

3. 新增 POST /feed/discovery 接口专门发布发现模块帖子
4. 执行 SQL 更新旧数据：`UPDATE t_feed_post SET post_type = 'TIMELINE' WHERE post_type IS NULL`

### 修复6：关注/粉丝列表点击跳转问题

**原因：** 内联表达式中的多个语句没有正确执行

**修复：** 创建专门的方法处理跳转
```typescript
function goToUserProfile(userId: number) {
  showFollowing.value = false
  showFollowers.value = false
  router.push(`/profile/${userId}`)
}
```

### 修复7：编译错误 - UserLevelInfo 参数不匹配

**原因：** UserLevelInfo record 定义需要4个参数，但调用时只传了3个

**修复：**
```java
return new UserLevelInfo(0, 1, 0, 50);  // 添加 scoreToNext 参数
```

### 修复8：关注/粉丝列表显示当前用户数据而非查看用户数据

**原因：** `loadProfile()` 函数只在 `isMe.value` 为 true 时才加载关注/粉丝数据，导致查看其他用户时显示当前用户的数据或显示0

**修复：**

**原因：** `handleOpenFollowing` 和 `handleOpenFollowers` 函数直接显示"功能开发中"，没有检查用户是否为管理员

### 修复10：TypeScript 编译错误

**原因：** 数组访问可能返回 undefined，split 结果可能为 undefined

   ```sql
   UPDATE t_user SET is_admin = 1, user_level = 10, activity_score = 3000 WHERE email = 'your_email@example.com';
   ```

2. 执行脚本：
   ```bash
   mysql -u root -pcampus123 campus_love < campus-love-backend/src/main/resources/db/set_admin.sql
   ```

```bash
mysql -u root -pcampus123 campus_love
```

然后执行：
```sql
-- 设置指定用户为管理员
UPDATE t_user SET is_admin = 1, user_level = 10, activity_score = 3000 WHERE email = 'your_email@example.com';

SELECT id, email, nickname, is_admin, user_level, activity_score FROM t_user WHERE is_admin = 1;
```
